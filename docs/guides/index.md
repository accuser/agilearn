# Python Guides

Fourteen guides, each structured the same way: **Learn** for tutorials, **Recipes** for task-focused how-tos, **Reference** for lookups, and **Concepts** for short explanatory essays. Every tutorial's code cells can be edited and run directly on the page.

## A recommended order

If you're working through Python for the first time, a sensible sequence is:

1. [Conditional logic](conditional-logic/) — how programs make decisions.
2. [Data structures](data-structures/) — lists, tuples, dictionaries, sets.
3. [Functions](functions/) — defining, calling, and composing.
4. [Classes and objects](classes-and-objects/) — bundling state and behaviour.
5. [String processing](string-processing/) — working with text.
6. [Dates and times](dates-and-times/) — `datetime`, parsing, and time zones.
7. [Type hints](type-hints/) — annotations and static type-checking.
8. [Iterators and generators](iterators-and-generators/) — lazy pipelines and the iteration protocol.
9. [File handling](file-handling/) — reading and writing data.
10. [Error handling](error-handling/) — exceptions and failure modes.
11. [Logging and debugging](logging-and-debugging/) — making programs observable.
12. [Regular expressions](regex/) — pattern matching for text.
13. [Unit testing](unit-testing/) — proving your code works.
14. [Packages and packaging](packages-and-packaging/) — modules, imports, virtual environments, and shipping your code.

If you're here for a specific topic, pick any guide and dive in — they're self-contained.
